# Инструкция для работы с Git

## Что такое Git 
Git - это одна из реализаций распределенных систем контроля версий, имеющая как и локальные, так и удаленные репозитории. Является самой популярной реализацией систем контроля версий в мире. 
## Основные команды для работы с Git 
* git init - создание репозитория
* git status - просмотр состояния репозитория
* git add .\имя файла - добавление изменений в коммит
* git commit -m "Сообщение" - создание (сохранение) коммита
* git log - просмотр журнала
* git branch - просмотр всех созданных веток
* git checkout - переход между сохраненными ветками
* git merge - команда для добавления содержимого одной ветки в другую
## Правила оформления и списки 
1. Для того, чтобы написать текст курсивом, нужно обрамить его звездочками (*) или подчеркиванием (_). Например: *курсив*, _курсив_.
2. Для того, чтобы написать текст полужирным, нужно обрамить его двумя звездочками (**) или двумя подчеркиваниями (__). Например: **полужирный**, __полужирный__.
3. Для того, чтобы написать текст полужирным курсивом, нужно обрамить его одной звездочкой и одним подчеркиванием (*_). Например: **_полужирный курсив_**.

## Работа с GitHub

GitHub сервис компании Майкрософт для организации работы с удаленными репозиториями. 

Основные команды:  

* git clone - создает копию удаленного репозитория
* git push - отправляет свою версию репозитория во внешний репозиторий
* git pull - скачать все из текущего репозитория и сделать merge c нашей версией
## Заключение 
Данная инструкция позволит получить базовые знания для создания репозитория и  внесение изменений в него.
